Static + Cron Setup

The website stays static; Render Cron handles scheduled email.

Dashboard Setup

Why Two Render Services?

A static website does not sleep, but it cannot safely hold a Resend API key or run a daily scheduler. The scheduled email must run in a private Render Cron Job.

Static site = always available public page. Cron Job = private scheduled sender.

Static Site Settings

  1. Create a new Render Static Site.
  2. Use the same GitHub repo.
  3. Set Build Command to blank.
  4. Set Publish Directory to public.
  5. Deploy.

Cron Job Settings

  1. Create a new Render Cron Job.
  2. Use the same GitHub repo.
  3. Use this command:
python cron_send_briefing.py

Add these environment variables to the Cron Job:

RESEND_API_KEY=re_your_key
RESEND_FROM_EMAIL=Market Briefing <onboarding@resend.dev>
BRIEFING_RECIPIENT_EMAIL=xin.chen.29@gmail.com
BRIEFING_SYMBOLS=^GSPC,^IXIC,^DJI,^VIX,ES=F,NQ=F,CL=F,GC=F,EURUSD=X,BTC-USD
BRIEFING_HEADLINE_COUNT=8
BRIEFING_TIMEZONE_LABEL=America/New_York
BRIEFING_SUBJECT_PREFIX=Market Morning Briefing

Schedule

Set the Cron Job schedule to the morning time you want. Render Cron schedules are configured in Render, not in the static website.

For example, for 8:00 AM Eastern, choose a daily schedule for 8:00 AM in Render if it offers timezone selection. If it asks for UTC, use 12:00 UTC during EDT or 13:00 UTC during EST.
